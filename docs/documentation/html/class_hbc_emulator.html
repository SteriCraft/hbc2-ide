Detailed Description

Where the HBC-2 emulation is done.

This class is responsible for emulating the HBC-2 CPU and its surrounding components. It uses a separate thread to increase performance.
It uses a thread safe struct (Emulator::Status) to be controlled by the main thread.

List of emulator states
ID State Description
0 NOT_INITIALIZED No binary data was provided, the thread is not running
1 READY Just after binary data was provided, the thread is executed and waiting for Command::RUN
2 RUNNING Emulator running at the targeted frequency
3 PAUSED Emulator paused during execution
List of available commands
ID Command Description Keyboard shortcut
0 NONE No operation
1 RUN Starts execution at targeted frequency and sets the emulator's state to RUNNING F9
2 STEP Executes one instruction, does not change the emulator's state F10
3 PAUSE Pauses execution and sets the emulator's state to PAUSED F11
4 STOP Stops execution and sets the emulator's state to READY F12
5 CLOSE Stops the thread and sets the emulator's state to NOT_INITIALIZED

Any invalid command will result in NONE.
