Detailed Description

Derived from HbcPeripheral, represents the EEPROM device.

This class is responsible for emulating the HBC-2 EEPROM.

Device ID: 0xE1

It stores 1 MiB of data (20-bit address).

On startup, it will dump in RAM:

  1. the IVT (0x0100 - 0x02FF),
  2. the 512 first bytes of program memory (0x0300 - 0x04FF),
  3. the interrupt handlers (0xF000 - 0xFFFF).

Port ADDR_0 is the most significant byte of the address.

If the given address is larger than 20 bits (3 bytes = 24 bits), the 4 most significant bits are ignored.

Control

Like every HbcPeripheral, HbcEeprom uses sockets connecting it to HbcIod ports to communicate with HbcCpu.

List of available ports
ID Port Description
0 CMD Command sent by HbcCpu
1 DATA Data read or written
2 ADDR_0 Most significant byte of the 20-bit address (only the 4 least significant bits are relevant)
3 ADDR_1 Middle byte of the 20-bit address
4 ADDR_2 Least significant byte of the 20-bit address
List of available commands
ID Command Description
0 NOP No operation
1 READ Reads the byte at the given address on DATA port
2 WRITE Writes the byte on DATA port at the given address

Any invalid command will result in NOP.
