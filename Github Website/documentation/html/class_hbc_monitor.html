Detailed Description

Derived from HbcPeripheral, represents the Monitor device.

This class is responsible for emulating the HBC-2 monitor. It uses a separate thread to increase performance.
It uses a thread safe struct (Monitor::Status) to be controlled by the main thread.

Color palette

HbcMonitor uses a 4-bit color palette.

Video modes
ID Mode Description
0 PIXEL 256 * 192 px 4-bit color mode
1 TEXT 6 * 8 px characters in a 42 columns by 24 rows grid, each with a 4-bit color background and a 4-bit color font

Colored text mode

Displays a 42 columns by 24 rows grid of characters.
95 characters can be displayed, corresponding to ASCII codes 32 to 127 (included).

Each character can have a 4-bit color background, and another 4-bit color font.

Pixel mode

Displays the whole 256 * 192 pixels monitor with 4-bit colors (see the palette above).
In this mode, each byte of video memory represented 2 pixels.
This would use 2,016 bytes of memory, but for simplification this code stores each pixel in a whole byte.

Control

Like every HbcPeripheral, HbcMonitor uses sockets connecting it to HbcIod ports to communicate with HbcCpu.

List of available ports
ID Port Description
0 DATA_0 First data byte, used for color information in both video modes
1 DATA_1 Second data byte, used for colored text mode only
2 POS_X X-axis position of the designated pixel or character
3 POS_Y Y-axis position of the designated pixel or character
4 CMD Command sent by HbcCpu
List of available commands
ID Command Description
0 NOP No operation
1 WRITE Changes pixel or character information on DATA_0 and DATA_1 ports, depending on POS_X and POS_Y
2 READ Sends pixel or character information on DATA_0 and DATA_1 ports, depending on POS_X and POS_Y
3 SWITCH_TO_PIXEL_MODE Switches to pixel mode
WARNING: Does not flush the character buffer
4 SWITCH_TO_TEXT_MODE Switches to colored text mode
WARNING: Does not flush the pixel buffer

Any invalid command will result in NOP.
